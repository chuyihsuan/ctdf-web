五階段開發規劃

依 FRD / PRD / SRD，第一階段先完成可公開審閱的開發環境與靜態 Demo。

1. 環境與 Demo

GitHub 專案、靜態首頁、Vercel / GitHub Pages 審閱流程、無個資示意資料。

2. 前台 MVP

新聞、賽事、排名、選手、關於、文件與公平競技等可瀏覽頁面。

3. Supabase 資料庫

核心資料表、測試資料、公開讀取規則與未來後台串接準備。

4. 後台管理

登入、角色權限、內容管理、賽事排名維護與操作紀錄。

5. 正式上線準備

官方網域、SSL、RLS、備份、SEO、教育訓練與上線檢查。

第一階段路由方向

靜態 Demo 先展示資訊架構；完整路由會在第二階段前台 MVP 實作。
